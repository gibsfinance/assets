### Performance Analysis Completed

**Top bottlenecks identified:**
- Etherscan Puppeteer: 83+ seconds per chain, fully serialized
- Image fetching: no parallelism, 4-5 DB round-trips per token (now has TTL caching)
- Retry strategy: constant 10s delays, 5 attempts = 40s per failed fetch
- DexScreener BFS: unbounded traversal, but needed for full coverage

**New utilities available for collectors:**
- `storeToken` — for collectors that don't need images (routescan already using it)
- `insertTokenBatch` — for batch-inserting tokens (not yet wired into collectors)
- `resolveImage` — for separating image fetch from DB writes

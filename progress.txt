**Design System:**

**Full-Page Physics Animation:**
- 60-80 floating token/network icons on a fixed-position `<canvas>`
- Collision physics: damped elastic bounces, mass proportional to icon size
- Spatial hash grid for O(n) collision detection (only within same depth layer)
- Wall bounces, scroll force, mouse repulsion ("parting the waters" effect)
- 3 depth layers: background (20-30px), middle (40-60px), foreground (60-80px)
- 4% chance of "monster" icon (168px)
- `prefers-reduced-motion` fallback to static icons
- Engine in `lib/physics/` (types.ts, engine.ts, forces.ts) — pure functions, testable

### Server API endpoints (used by frontend)
- `GET /list` — all available lists
- `GET /list/{providerKey}/{listKey}?chainId={id}` — specific list
- `GET /networks` — supported networks
- `GET /image/{chainId}` — network icon
- `GET /image/{chainId}/{address}` — token icon (priority-ordered)
- `GET /image/direct/{hash}` — direct by hash
- `GET /image/fallback/{order}/{chainId}/{address}` — with explicit order (comma-separated provider keys)

### Key design decisions
- Tailwind CSS 4 @theme for design tokens — raw utilities + Headless UI for primitives
- Hash-based routing (#/studio, #/docs) — no server required for routing
- localStorage caching with 3hr TTL and chunking for large token lists
- No web3/wallet integration — pure read-only asset browser
- Static SPA build deployed to Railway
- Physics engine as pure functions in lib/physics/ — reusable, testable
- Badge position as polar coordinates (angle + overlap) → CSS absolute positioning
- StudioContext owns cross-page chain pre-selection via localStorage
